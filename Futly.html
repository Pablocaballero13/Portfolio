El proyecto

Por qué existe Futly

El fútbol base en España mueve miles de equipos de formación. Cada uno coordina entrenador, jugadores y familias... principalmente a través de grupos de WhatsApp.

Las convocatorias se pierden entre memes, no hay historial de estadísticas y no existe ningún canal oficial. Futly centraliza todo eso en una plataforma pensada específicamente para esto.

Sistema completamente funcional y desplegado en producción, con registro por roles, gestión de partidos y convocatorias, actas con estadísticas y avisos entre entrenador y familias.

API REST

Endpoints principales

POST /auth/registro/entrenador Registro + creación automática de equipo
POST /auth/login Login → devuelve JWT + rol + equipo_id
GET /jugadores/equipo/{equipoId} Lista jugadores del equipo
POST /partidos/{id}/convocar Convoca jugador + genera aviso automático
DEL /partidos/{id}/convocar/{jid} Desconvoca jugador del partido
POST /partidos/{id}/acta Guarda acta con estadísticas individuales
GET /estadisticas/jugador/{id} Goles, asistencias, minutos y partidos jugados
POST /avisos/ausencia Padre reporta ausencia al entrenador
PATCH /avisos/{id}/revisar Marca aviso como revisado
PUT /auth/perfil Edición de nombre, email o contraseña
Seguridad

Diseñado para
ser seguro

🔐

Autenticación JWT

Al hacer login se genera un token firmado con PyJWT. Cada petición a endpoints protegidos lo valida automáticamente. Token inválido = 401.

🔒

Contraseñas con bcrypt

Nunca se guardan en texto plano. bcrypt aplica un salt aleatorio por contraseña, evitando que dos iguales produzcan el mismo hash.

👮

Autorización por roles

Cada endpoint sensible comprueba el rol en el servidor. Un padre intentando crear un partido recibe directamente un 403 Forbidden.

📱

EncryptedSharedPreferences

El token JWT en el dispositivo Android se cifra con AES256-GCM antes de guardarse. Si alguien accede al dispositivo, no puede leerlo.
