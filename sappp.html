 2. Explicación de la lógica técnica (ABAP)

            
            

                
01

                

                    
Conexión HTTP Externa (cl_http_client)

                    
Para comunicar SAP con el exterior de manera segura, invoqué la clase estándar cl_http_client=>create_by_url. Se mapean las cabeceras REST, fijando el método en POST, el Content-Type en application/json e inyectando de forma asíncrona la API Key del modelo.

                

            


            

                
02

                

                    
Estructuración del Prompt para forzar un JSON estricto

                    
Para evitar las respuestas conversacionales por parte de la IA (como "Claro, aquí tienes tu respuesta:"), se diseña un *system prompt* restrictivo que obliga al LLM a devolver **únicamente** la estructura JSON cruda, permitiendo un procesamiento seguro en el servidor SAP.

                

            


            

                
03

                

                    
Parseo optimizado con Expresiones Regulares (REGEX)

                    
Al procesar la respuesta, en lugar de instanciar pesados parsers orientados a objetos que consumen memoria de diálogo, utilicé sentencias nativas de ABAP:

                    
FIND REGEX '"content"\s*:\s*"(.*?)"' IN iv_raw_json SUBMATCHES lv_content.

                    
De esta manera, aislo la cadena del mensaje, limpio los caracteres de escape y vuelvo a utilizar bloques SUBMATCHES para asignar los valores directamente a los campos de la estructura interna del ERP.
